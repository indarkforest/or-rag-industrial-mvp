# Embedding 向量化知识笔记

## 一、什么是 Embedding

把一段文本转换成一组浮点数（向量），这组数就是文本在语义空间中的"坐标"。语义相近的文本，向量距离也近。

```
"反应釜温度过高"  →  [0.12, -0.34, 0.56, ..., 0.78]  ← 1536 个数
"釜温超限"        →  [0.11, -0.32, 0.55, ..., 0.77]  ← 很接近
"冷却水流量正常"  →  [-0.45, 0.21, -0.33, ..., 0.02] ← 差别大
```

---

## 二、为什么语义相近的文本向量距离近

### 训练原理

Embedding 模型（如 MiniMax embo-01）在**海量语料**上训练，学会了把"意思相近"的文本映射到向量空间的同一区域。

### 训练过程（简化版）

**第一步：收集海量文本对**

训练数据中有大量这样的配对：

```
("反应釜温度过高", "釜温超限")        ← 语义相近，标注为"正样本"
("反应釜温度过高", "冷却水流量正常")   ← 语义无关，标注为"负样本"
("TI-101温度上升", "测点温度升高")     ← 语义相近
("TI-101温度上升", "搅拌器转速120rpm") ← 语义无关
...（billions of pairs）
```

**第二步：训练神经网络**

```
                    神经网络（Transformer）
                         ┌──────────┐
文本A ──────────────────→│          │──→ 向量A [0.12, -0.34, ...]
                         │  embo-01 │
文本B ──────────────────→│          │──→ 向量B [0.11, -0.32, ...]
                         └──────────┘
                              │
                    损失函数: 让正样本的向量A、B靠近
                              让负样本的向量A、B远离
```

训练目标用数学表达：

```
正样本（语义相近）:  最大化 cos(A, B)  →  趋近 1
负样本（语义无关）:  最小化 cos(A, B)  →  趋近 0
```

经过数十亿次的调整，神经网络学会了：**把语义相近的文本输出到相近的向量位置**。

## 三、维度与向量空间

### 为什么是 1536 维

这是 MiniMax `embo-01` 模型的设计决定。每个维度捕捉一种**语义特征**，1536 个维度组合起来就能精确区分任意两段文本的语义关系。

### 维度相同才能在同一评判体系比较

```
1536 维的向量  ←→→  1536 维的向量空间  ←→→  可以计算相似度
1024 维的向量  ←→→  1024 维的向量空间  ←→→  与 1536 维不在同一空间，无法比较
```

这就是为什么更换 embedding 模型后**必须重新 build 知识库**——新模型和旧库的向量不在同一空间，维度不同，无法做矩阵点积。

### 高维空间的数学合法性

线性代数中，维度增加不会改变任何公式，只是坐标更多：

| 维度 | 几何直觉 | 距离公式 |
|---|---|---|
| 2 维 | 平面上两点距离 | `√((x₁-x₂)² + (y₁-y₂)²)` |
| 3 维 | 空间中两点距离 | `√((x₁-x₂)² + (y₁-y₂)² + (z₁-z₂)²)` |
| 1536 维 | 无法想象，但公式一样 | `√(Σ(xᵢ-yᵢ)²)` |

所有线性代数运算（点积、矩阵乘法、正交投影、特征值分解）在高维空间中**公式完全相同**，只是计算量更大。

## 四、文本在向量空间中的表示

### 核心流程

```
第 1 步: 模型训练
  ┌──────────────────────────────────┐
  │ 海量文本 → 神经网络 → 1536维空间  │
  │                                    │
  │ 训练目标:                          │
  │   语义相近的文本 → 空间中靠近       │
  │   语义无关的文本 → 空间中远离       │
  │                                    │
  │ 结果: 1536个轴自动形成，不可解释    │
  └──────────────────────────────────┘

第 2 步: 文本向量化
  "TI-101温度过高"
       │
       ▼ 神经网络计算
  [0.12, -0.34, 0.56, ..., 0.78]  ← 1536个坐标值
       │
       ▼ 就是1536维空间中的一个点

第 3 步: 相似度计算
  点A "TI-101温度过高"    [0.12, -0.34, 0.56, ...]
  点B "釜温超限"          [0.11, -0.32, 0.55, ...]
       │
       ▼ 算两个点的夹角
  cos(θ) = 0.92  ← 夹角很小 → 语义相近

  点A "TI-101温度过高"    [0.12, -0.34, 0.56, ...]
  点C "冷却水流量正常"    [-0.45, 0.21, -0.33, ...]
       │
       ▼ 算两个点的夹角
  cos(θ) = 0.15  ← 夹角大 → 语义无关
```

## 五、语义相似度计算

### 余弦相似度

```
sim(A, B) = (A · B) / (|A| × |B|)

其中:
  A · B = Σ(aᵢ × bᵢ)        ← 点积（对应位相乘再求和）
  |A|   = √(Σaᵢ²)           ← 向量模长
  |B|   = √(Σbᵢ²)
```

几何含义就是两个向量的**夹角余弦值**：

```
        A向量
       ↗
     / ← 夹角越小，越相似
   /________ B向量

夹角 ≈ 0°  →  sim ≈ 1.0  →  语义相同
夹角 ≈ 90° →  sim ≈ 0.0  →  无关
夹角 ≈ 180°→  sim ≈ -1.0 →  语义相反
```

### 归一化 = 投影到单位球面

归一化后所有向量长度变为 1，点积直接等于余弦值：

```
归一化前: cos(θ) = (A·B) / (|A|×|B|)    ← 需要除以模长
归一化后: cos(θ) = A·B                   ← 模长为1，直接点积
```

### 矩阵乘法 = 批量相似度计算

```
Matrix (N×1536)  @  Vector (1536,)  =  Scores (N,)

┌─────────────────────┐     ┌────┐     ┌─────┐
│ v1: [0.1, 0.3, ...] │     │    │     │ 0.87 │ ← v1与查询的相似度
│ v2: [0.2, 0.1, ...] │  ×  │ q  │  =  │ 0.42 │ ← v2与查询的相似度
│ v3: [0.8, 0.5, ...] │     │    │     │ 0.91 │ ← v3与查询的相似度
│ ...                 │     │    │     │ ...  │
└─────────────────────┘     └────┘     └─────┘
```

代码中 `matrix @ query_vec` 一行就是线性代数的矩阵-向量乘法。

## 六、Embedding 与线性代数的对应关系

| 线性代数概念 | 在 Embedding 中的体现 |
|---|---|
| **向量** | 一段文本的 1536 维表示 `[0.12, -0.34, ...]` |
| **向量空间** | 所有文本被映射到的 1536 维空间 |
| **基向量** | 1536 个坐标轴，每个轴代表一种语义特征 |
| **向量长度（模）** | 文本信息的"强度" |
| **夹角（余弦）** | 两段文本的语义相似度 |
| **点积** | 余弦相似度的计算核心 |
| **矩阵乘法** | 批量计算查询向量与所有库向量的相似度 |
| **正交/线性无关** | 语义独立性 |

Embedding 本质上就是**用线性代数的工具，把"语义"这个抽象概念变成可计算的数学对象**。

## 七、理解与指正

### ✅ 正确的理解

| 理解 | 状态 |
|---|---|
| 维度相同才能在同一评判体系比较 | ✅ 完全正确 |
| 1536 维和 1024 维不是同一空间，无法比较 | ✅ 完全正确 |
| 每个轴代表一个概念方向 | ✅ 大致正确 |
| 文本被语义理解后算出每个维度的坐标 | ✅ 正确 |
| 文本在向量空间中是一个点 | ✅ 完全正确 |
| 多维度相近 → 关系越强 | ✅ 正确 |

### ⚠️ 需要修正的理解

**修正 1：轴的含义不是"时间""空间"这种人类概念**

设想中的"第 a 维度是时间，第 b 维度是空间"是人类能理解的标签，但实际 embedding 模型训练出来的 1536 个轴是**不可解释的**：

```
设想:
  第 1 轴 = 温度
  第 2 轴 = 压力
  第 3 轴 = 因果关系
  ...

实际:
  第 1 轴 = ???（模型自己学的，无法用人类语言描述）
  第 2 轴 = ???（可能是"温度×设备"的某种混合特征）
  第 3 轴 = ???（可能是"数值大小×工业领域"的组合）
  ...
```

模型通过神经网络自动学出了 1536 个特征轴，但这些轴是**复合的、纠缠的**，不能简单对应到人类概念。就像人脑的神经元——你能识别"猫"，但找不到哪个神经元专门负责"猫"。

**修正 2：不是"某个轴相近 = 某个概念相似"**

```
设想:
  "温度过高" 和 "釜温超限" 在"温度轴"上坐标相近 → 温度相似

更准确的理解:
  "温度过高" 和 "釜温超限" 在 1536 个轴上整体坐标都相近
  → 综合相似度高（余弦相似度 ≈ 0.92）

  但无法说"它们在第 127 轴上相近所以是温度相似"
  因为第 127 轴可能同时编码了温度+工业+报警三种语义
```

**修正 3："越多维度相近 → 关系越强"需要补充**

不是"相近维度的**数量**"，而是"所有维度的**综合距离**"：

```
情况 A: 1536 维中有 1000 维完全相同，536 维差异极大
情况 B: 1536 维中每一维都有微小差异，但没有完全相同的

情况 B 的综合相似度可能更高
→ 因为余弦相似度看的是整体夹角，不是逐维匹配计数
```

类比：两个人在 3D 空间中的位置——判断他们"近不近"是看**直线距离**，不是数"有几个坐标轴上的值相同"。

## 八、在本项目中的实际应用

### 构建知识库时

```
事实块文本: "温度测点（TI-101）的监测对象是聚合温度，触发报警是TAH-101。"
     │
     ▼ MiniMax embo-01 API (type=db)
向量: [0.23, -0.15, 0.67, ..., 0.42]  ← 1536 个 float32
     │
     ▼ 存入 SQLite facts 表 (embedding BLOB 字段)
二进制: 1536 × 4 bytes = 6144 bytes
```

### 查询时

```
问题: "TI-101 温度高了会怎样？"
     │
     ▼ MiniMax embo-01 API (type=query)
向量: [0.21, -0.13, 0.65, ..., 0.40]  ← 1536 个 float32
     │
     ▼ 与 facts 表所有向量做余弦相似度（矩阵乘法）
排序 → 取 top 6 个最相似的事实块
```

### 之前的问题 vs 修复后

| | HashingVectorizer (1024维) | embo-01 (1536维) |
|---|---|---|
| **原理** | 字符 n-gram 哈希 | 神经网络编码语义 |
| **"温度高" vs "超温"** | 不匹配（字符不同） | 匹配（语义相同） |
| **"冷却水" vs "chilled water"** | 不匹配 | 匹配 |
| **训练数据** | 无（纯算法） | 海量语料训练 |
| **输出** | 稀疏二值向量 | 稠密浮点向量 |

这就是修复后 OG-RAG 检索质量大幅提升的根本原因。

## 九、一句话总结

**文本 → 高维空间中的点 → 点之间距离近 = 语义相近**。不需要纠结每个轴具体代表什么——它是一个整体性的、不可拆分的语义空间，我们只关心点与点之间的**综合距离**。
